### 서울 - Opening Paragraph

라운드HR이 **다단계 서류 평가 액션** 기능을 출시했습니다. 이 기능을 통해 채용 담당자는 1차/2차 서류 평가자와 평가표를 미리 설정해두면, 1차 평가 완료 시 자동으로 2차 평가자가 추가되고 알림이 발송됩니다.

기존에는 채용 담당자가 1차 평가 후 슬랙이나 메일로 일일이 2차 평가자에게 알려야 했습니다. 이로 인해 HR팀은 채용 시즌마다 반복적인 수동 작업에 시간을 빼앗기고, 알림 누락으로 평가 지연이 발생하기도 했습니다.

#### Problem Statement

> "1차 서류 검토 끝나면 슬랙으로 팀장님한테 일일이 알려드려야 해요. 팀장님이 출장 중이시면 슬랙 확인이 늦어서 2~3일 지연되기도 하고요. 여기에 '이 사람은 왜 통과시켰어요?' 물어보시면 또 설명해드려야 해요. 업무 시간의 30%는 이런 일에 쓰는 것 같아요."
> — 페이멘트랩 HR팀 리드

현재 라운드HR은 서류 평가 단계가 **1개만 지원**되고, 평가도 **1개만 추가** 가능합니다. 이로 인해:

- HR팀(1차)과 현업(2차)이 순차적으로 평가하는 고객은 **수동 워크플로우**에 의존
- 1차 평가 결과(코멘트)가 2차 평가자에게 **공유되지 않아** 반복 설명 필요
- 알림 타이밍을 제어할 수 없어 **불필요한 알림** 또는 **알림 누락** 발생

#### Solution

**다단계 서류 평가 액션**은 과제 요청, 레퍼런스 체크와 동일한 **"액션"** 패턴으로 구현되어, 채용 담당자가 사전에 1차/2차 평가자와 평가표를 설정해두면 트리거 조건에 따라 자동으로 실행됩니다.

**핵심 가치:**
- **제로 수동 알림**: 1차 평가 완료 시 자동으로 2차 평가자 추가 + 알림 발송
- **평가 맥락 공유**: 2차 평가자가 1차 평가 코멘트를 바로 확인
- **유연한 조건 설정**: 점수 기준, 결과 기준, 다중 평가자 AND 조건 지원

#### 주요 기능

**1. 평가자 및 평가표 사전 설정 (MVP)**
- 1차 평가자 + 평가표 지정
- 2차 평가자 + 평가표 지정
- 공고 단위로 한 번만 설정하면 모든 지원자에 적용

**2. 트리거 조건 설정 (MVP)**
- 1차 평가 결과 = 합격 시 2차 진행
- 1차 평가 점수 ≥ N점 시 2차 진행
- 1차 평가자 전원 합격(AND) 시 2차 진행

**3. 평가 코멘트 자동 공유 (MVP)**
- 2차 평가자가 1차 평가 코멘트를 지원자 상세에서 바로 확인
- "왜 이 사람 통과시켰어요?" 질문 불필요

**4. 서류 평가 자동화 확장 (Phase 2)**
- 1차 평가자 A, B 각각의 결과 조합에 따른 자동화 실행
- 예: 둘 다 합격 → 2차 진행, 둘 중 하나만 합격 → 보류, 둘 다 불합격 → 불합격

#### 차별화

기존 ATS는 서류 평가 단계를 물리적으로 분리하여 구현합니다. 이는 DB 스키마 변경, 마이그레이션, 기존 고객 영향 등 큰 비용이 수반됩니다.

라운드HR의 **다단계 서류 평가 액션**은:
- 시스템상 서류 평가 단계는 **1개를 유지**하면서
- **액션 패턴**을 활용해 평가자와 평가표를 자동 추가
- 기존 고객 영향 없이 **점진적 확장 가능**

#### 고객 추천사

**차나린 매니저 (게임듀오, 게임):**
> "전에는 a 평가자에게 먼저 요청하고 b에게 나중에 요청하고 싶어도 방법이 없었어요. 이제는 설정만 해두면 1차 끝나고 자동으로 2차에 알림이 가니까 정말 편해요."

**기어세컨드 채용팀:**
> "2차 평가자가 1차 평가 코멘트를 확인할 수 없어서 매번 설명해야 했는데, 이제는 코멘트가 바로 보여서 평가 품질도 올라갔어요."

**김수현 HR팀 리드 (페이멘트랩, 핀테크):**
> "슬랙으로 일일이 알림 보내던 시간이 사라졌어요. 한 달에 3~4개 공고 열면 매번 반복하던 작업이었는데, 이제는 설정 한 번으로 끝이에요. 업무 시간 30%는 줄어든 것 같아요."

#### 측정 가능한 결과

Beta 테스트 고객이 달성한 것:
- 수동 평가자 추가/알림 작업 **80% 감소**
- 1차 → 2차 평가 시작까지 소요 시간 **2~3일 → 즉시**
- 평가 맥락 공유로 인한 **반복 설명 시간 30분~1시간 절감** (채용 건당)

## FAQ

### Product & Features

#### Q1. 이 기능은 어떻게 작동하나요?

**A1.** 공고 설정 > 자동화 > 서류 평가 액션에서 다음을 설정합니다:

1. **1차 평가 설정**: 평가자 + 평가표 지정
2. **트리거 조건**: 1차 평가 결과=합격, 점수≥N점, 전원 합격(AND) 중 선택
3. **2차 평가 설정**: 평가자 + 평가표 지정

지원자가 서류 단계에 진입하면 1차 평가자에게 자동 요청되고, 1차 평가 완료 시 트리거 조건을 체크하여 조건 충족 시 2차 평가자와 평가표가 자동 추가됩니다.

---

#### Q2. 과제 요청, 레퍼런스 체크와 어떻게 다른가요?

**A2.** 동일한 "액션" 패턴을 사용합니다.

| 기능 | 트리거 | 액션 |
|:---|:---|:---|
| 과제 요청 | 특정 단계 진입 시 | 과제 양식 자동 발송 |
| 레퍼런스 체크 | 특정 단계 진입 시 | 레퍼런스 요청 자동 발송 |
| **다단계 서류 평가** | 1차 평가 완료 시 | 2차 평가자+평가표 자동 추가 |

---

#### Q3. 3차 이상의 평가도 지원하나요?

**A3.** MVP에서는 1차 → 2차까지만 지원합니다. 향후 고객 피드백에 따라 N차 확장을 검토할 예정입니다.

---

### User Experience & Workflow

#### Q4. 기존에 진행 중인 채용 공고에도 적용할 수 있나요?

**A4.** 네, 기존 공고에도 설정 가능합니다. 다만, 이미 서류 단계를 지난 지원자에게는 소급 적용되지 않습니다.

---

#### Q5. 1차 평가 완료 전에 2차 평가자에게 알림이 가지는 않나요?

**A5.** 아니요. 트리거 조건(1차 평가 완료)이 충족되어야만 2차 평가자에게 알림이 발송됩니다.

**중복 알림 방지 로직:**
- 지원자별로 액션 실행 상태를 저장
- 이미 실행된 경우 재실행되지 않음

---

#### Q6. 2차 평가자가 1차 평가 코멘트를 어디서 확인하나요?

**A6.** 지원자 상세 페이지 > 평가 탭에서 확인할 수 있습니다. 1차 평가자의 점수와 코멘트가 함께 표시됩니다.

### Technical & Implementation

#### Q9. 기존 자동화 설정(점수 기반 단계 이동)과 충돌하지 않나요?

**A9.** 독립적으로 동작합니다. 기존 `AutoActionReview`(점수 기반 단계 이동)와 신규 `서류 평가 액션`(2차 평가자 추가)은 별개의 트리거로 작동하며, 둘 다 활성화할 수 있습니다.

---

#### Q10. 중복 알림이 발송되는 경우는 없나요?

**A10.** 없습니다. `candidate_auto_actions` 테이블에 실행 기록을 저장하여, 동일 지원자에 대해 액션이 1회만 실행되도록 보장합니다.
