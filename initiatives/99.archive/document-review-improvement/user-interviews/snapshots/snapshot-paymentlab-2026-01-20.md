# 인터뷰 스냅샷: 페이멘트랩 김수현

**날짜:** 2026-01-20  
**유형:** 모의 인터뷰 (페르소나 기반)  
**소요 시간:** 15분  
**인터뷰어:** Terry

---

## 주요 사실
- **세그먼트:** HR팀 리드 / 핀테크 스타트업 (70명 규모)
- **주요 행동:** HR팀 1차 → 현업 팀장 2차 순차 서류 평가 진행
- **사용 도구:** 라운드HR, 슬랙, 개인 엑셀 메모
- **경험 수준:** 채용 담당 3년 이상
- **상황:** 월 3~4개 공고, 공고당 50명 내외 지원자

---

## 인상적인 인용문

> "1차 끝나고 팀장님한테 슬랙으로 일일이 알려드려야 했어요. 라운드에서 자동으로 알림 가면 좋겠는데, 그런 게 없으니까 제가 수동으로 요청한 거죠."

> "팀장님이 '이 사람은 왜 통과시킨 거예요?' 물어보시면 제가 기억을 더듬거나, 제 개인 메모를 뒤져서 다시 설명해드려야 해요."

> "1순위만 해결되면 제 업무 시간 30%는 줄어들 것 같아요."

---

## 스토리 요약

### 스토리 1: 백엔드 개발자 채용 (지난주)
**맥락:** 50명 지원자, HR팀 1차 → 개발팀장 2차 평가  
**발생한 일:**
1. 김수현이 50명 중 18명 1차 통과 처리
2. 슬랙으로 개발팀장에게 수동 알림
3. 팀장 출장 중으로 슬랙 확인 늦어 2~3일 지연
4. 팀장이 "왜 이 사람 통과?" 질문 → 구두로 다시 설명
5. 18명 중 7명 면접 대상자 확정

**결과:** 수동 알림 + 구두 설명으로 시간 소모

**주요 순간:**
- 슬랙 알림이 자동이었으면 지연 없었을 것
- 1차 평가 코멘트가 시스템에 없어서 반복 설명 필요
- 18명 설명에 30분~1시간 소요

---

## 경험 맵

**범위:** 1차 서류평가 완료 → 2차 평가자 평가 시작  
**목표:** 1차 결과 기반으로 2차 평가 빠르게 진행

| 단계 | 행동 | 감정 | 페인 포인트 |
|:---|:---|:---|:---|
| 1. 1차 평가 완료 | 엑셀에 개인 메모 작성 | 피곤함 | 시스템에 기록 안 됨 |
| 2. 2차 평가자 알림 | 슬랙으로 수동 요청 | 번거로움 | 자동 알림 없음 |
| 3. 2차 평가자 확인 지연 | 슬랙 확인 대기 | 불안함 | 타이밍 제어 불가 |
| 4. 1차 내용 설명 | 구두/슬랙으로 재설명 | 짜증남 | 코멘트 공유 안 됨 |
| 5. 진행 상황 추적 | 개별 클릭해서 확인 | 답답함 | 대시보드 없음 |

## 기회

### 🎯 Opportunity 1: 순차적 평가자 자동 요청
**Pain Point:** 1차 완료 후 슬랙으로 수동 알림  
**기대 효과:** 알림 업무 제거, 지연 방지

### 🎯 Opportunity 2: 평가 코멘트 공유
**Pain Point:** 2차 평가자가 1차 코멘트 확인 불가 → 반복 설명  
**기대 효과:** 설명 시간 30분~1시간 절감

### 🎯 Opportunity 3: 조건 기반 자동화
**Pain Point:** 탈락 확실한 사람도 2차에 넘어감  
**기대 효과:** 점수/결과 기준으로 자동 분기

### 🎯 Opportunity 4: 평가 현황 대시보드
**Pain Point:** 진행률 확인 위해 개별 클릭 필요  
**기대 효과:** 한눈에 현황 파악

### 🎯 Opportunity 5: 평가 기한 + 리마인더
**Pain Point:** 마감일 전달 어려움  
**기대 효과:** 자동 리마인더로 지연 방지

## 인사이트

### 평가 단계별 역할 분리
- **1차 (HR):** 기본 요건 스크리닝 (탈락 거르기)
- **2차 (현업):** 기술/역량 심층 검토 (면접 대상 선정)
- 두 단계의 평가 기준이 **완전히 다름**

### 정보 단절이 핵심 문제
- 1차 → 2차 간 **코멘트 공유 부재**가 가장 큰 비효율
- 평가 "점수"보다 "이유"가 더 중요한 정보

### 자동화 조건 니즈
- **점수 기준**: 70점 이상만 2차 요청
- **결과 기준**: 합격/보류/불합격으로 분기
- **수동 트리거**: "1차 완료" 버튼 클릭 시 요청

### 확산 가능성
- HR 커뮤니티에서 비슷한 고민 많음
- 50명 이상 스타트업, 체계화된 채용 프로세스 보유 회사 타겟

## 추가 발견 인사이트

- **3단계 평가 가능성:** 임원급/핵심 포지션은 HR → 팀장 → CTO 3단계
- **평가자 의견 충돌:** 2차 평가자가 2명일 때 조율 어려움
- **동일 지원자 히스토리:** 여러 공고 지원 시 이력 확인 어려움
- **모바일 평가:** 이동 중 서류 검토 니즈 (현재 불편)
