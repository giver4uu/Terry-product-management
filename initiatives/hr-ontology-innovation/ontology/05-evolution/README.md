# Phase 5: Evolution (진화 및 유지보수)

**목적:** 온톨로지를 지속적으로 개선하고 진화시킴

이 폴더는 **변경 이력, 버전 관리, 개선 백로그**를 포함합니다.

## 📁 포함 문서

- **`README.md`**: 진화 전략 (이 문서)
- **`change-log.md`**: 버전별 변경 이력
- **`versioning.md`**: 버전 관리 규칙 (Semantic Versioning)
- **`improvement-backlog.md`**: 개선 백로그

## 🔄 진화 원칙

**점진적 진화 (Iterative Evolution):**
- 완벽한 온톨로지를 한 번에 만들지 않음
- 파일럿 → 피드백 → 개선 사이클 반복

**버전 전략:**
```
v0.1.0: 폴더 구조 + 템플릿 (Week 1)
↓
v0.2.0: 7개 핵심 개념 + 파일럿 피드백 반영 (Week 7)
↓
v0.3.0: Problem 03, 04 개념 추가 (Week 12)
↓
v1.0.0: Production-ready, 3개 팀 사용 (3개월 후)
```

## 📊 진화 트리거

### 1. 인터뷰 기반 트리거
- 신규 인터뷰 (보리 추가, 제임스)
- 새 문제 발견
- 프로세스: 인터뷰 → 스냅샷 → CQ 업데이트 → 온톨로지 확장

### 2. 파일럿 기반 트리거
- 워크플로우 실패 (목표 시간 미달성)
- 사용성 문제 (이해도 < 4/5)
- 프로세스: 실패 분석 → Root Cause → 온톨로지 수정

### 3. 기술 발전 트리거
- 새 HR Tech 표준 (ESCO 업데이트)
- 업계 베스트 프랙티스 변화
- 프로세스: 외부 변화 모니터링 → 적용 가능성 평가 → 통합

### 4. 스케일 트리거
- 새 팀 추가 (팀 수 증가)
- 새 직무군 (Designer, Data Analyst)
- 프로세스: 확장성 평가 → 일반화 → 통합

## 🔄 변경 관리 프로세스

### Semantic Versioning
- **MAJOR (X.0.0)**: 기존 개념 삭제, 호환성 깨짐 → 모든 Stakeholder 승인 필요
- **MINOR (0.X.0)**: 새 개념/관계 추가 → Borry 승인 필요
- **PATCH (0.0.X)**: 정의 명확화, 예시 추가 → Terry 자체 검토

### 하위 호환성
- MINOR, PATCH는 반드시 하위 호환
- MAJOR는 Migration Guide 필수
