# Problem-Concept Mapping (문제-개념 매핑)

**목적:** 비즈니스 가치와 기술 솔루션의 명시적 연결

이 폴더는 **4개의 고객 문제가 온톨로지 개념으로 어떻게 해결되는지** 매핑합니다.

## 🎯 매핑의 목적

**"왜 이 개념이 필요한가?"에 대한 답변 제공**

### Before (매핑 없이)
```
개발자: "Competency 개념을 왜 만들어야 하죠?"
PM: "음... 중요하니까요?" ❌
```

### After (매핑 있음)
```
개발자: "Competency 개념을 왜 만들어야 하죠?"
PM: "problem-01-skill-standardization.md를 보세요.
     보리가 매 채용마다 2시간씩 JD를 처음부터 작성하는 문제를 해결합니다.
     Competency 정의가 표준화되면 30분으로 단축됩니다 (75% 개선)." ✅
```

## 📊 매핑 구조

각 매핑 문서는 다음을 포함합니다:

1. **문제 요약**
   - 고객 Pain Points (보리 인터뷰 기반)
   - 근본 원인
   - 비즈니스 임팩트 (정량적 지표)

2. **해결하는 온톨로지 개념들**
   - 핵심 개념 리스트 (어떤 Pain Point 해결하는지 명시)
   - 관계 리스트

3. **작동 방식 (How It Works)**
   - Before: 현재 문제 상황 (보리의 작업 흐름)
   - After: 온톨로지 사용 시 개선 상황
   - 기술 구현 예시 (JSON 구조)

4. **성공 지표 매핑**
   - 목표 지표 → 온톨로지 기여 → 측정 방법

5. **검증 계획**
   - Competency Questions 검증
   - 파일럿 테스트 시나리오

6. **우선순위 로드맵**
   - Phase 1-4 (Week별 계획)

## 🔗 Problem-Concept 연결 예시

### Problem 01: 스킬/역량 표준화
**해결하는 개념:**
- Competency (C-004): 스킬 표준 정의
- Proficiency Level (C-005): "Advanced"가 무엇인지 명확화
- Position (C-003): "시니어 백엔드 엔지니어"의 정확한 정의

**성공 지표:**
- JD 작성 시간: 2시간 → 30분 (75% ↓)
- 팀 간 합의: 1주일 → 1일 (85% ↓)

### Problem 02: 평가 기준 일관성
**해결하는 개념:**
- Evaluation Rubric (C-007): 5점 척도 with 행동적 앵커
- Competency (C-004): 평가 항목 표준 정의

**성공 지표:**
- 평가 조율 시간: 1시간 → 15분 (75% ↓)
- 면접관 간 신뢰도 향상: 20 percentage points
