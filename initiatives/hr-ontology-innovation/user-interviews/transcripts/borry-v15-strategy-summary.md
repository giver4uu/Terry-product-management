## 💡 V1.5 솔루션: 동적 데이터 추가

### 핵심 아이디어

**"ATS만이 가진 독점 자산 = 채용 과정 동적 데이터"**

일반 HR 온톨로지 (정적):
```
"Backend Engineer는 PostgreSQL Advanced가 필요하다"
```

우리 시스템 (정적 + 동적):
```
"Backend Engineer는 PostgreSQL Advanced가 필요하다"

  📊 과거 데이터 분석:
  → 지난 6개월 합격자 PostgreSQL 평균: 4.2/5
  → PostgreSQL 4점 미만은 모두 탈락 (15명)
  → 면접관 제임스는 PostgreSQL 평가 시 0.2점 관대

  💡 프로액티브 제안:
  "PostgreSQL 4점을 합격 기준으로 권장합니다"
```

**차이점:**
- Greenhouse: Borry가 추측으로 결정
- 우리: **시스템이 데이터 근거로 제안**

- [ ] 과거 채용 데이터 5-10건 정도 제공 가능하신가요?
