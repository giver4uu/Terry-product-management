# Opportunity: 팀 간 노하우 공유

**식별일:** 2025-11-16
**출처:** 보리 인터뷰 (snapshot-borry-2025-11-16.md)
**우선순위:** Medium-High
**임팩트 영역:** 조직 효율성, 채용 품질 균일화, 협업 문화

---

## 고객 문제 (Customer Problem)

### 구체적 상황
**개인과 팀의 우수한 채용 노하우가 조직 전체로 공유되지 않아**, 채용 품질이 담당자에 따라 들쑥날쑥하고 비효율이 반복됩니다.

### 실제 사례

**사례 1: 제임스의 암묵지**
- **상황:** 10년 경력의 개발 리드 제임스는 체계적인 면접 방식을 보유
- **문제:**
  - 제임스의 노하우는 문서화되어 있지 않음 (머릿속에만 존재)
  - 다른 면접관들은 제임스의 방법을 배울 수 없음
  - 제임스가 바빠서 면접에 불참하면 → 면접 퀄리티가 눈에 띄게 하락
- **결과:**
  - 채용 품질이 제임스 참여 여부에 의존
  - 제임스에게 과도한 부담
  - 다른 면접관들의 역량 성장 기회 상실

**사례 2: 백엔드 A팀 vs B팀의 단절**
- **상황:** 같은 회사에 백엔드 팀이 2개 존재
- **문제:**
  - A팀이 효과적인 면접 질문을 개발해도 B팀은 모름
  - 각 팀이 독립적으로 채용하며 정보 공유 시스템 부재
  - 같은 실수를 각 팀이 반복
- **결과:**
  - 바퀴의 재발명 (reinventing the wheel)
  - 팀 간 직무 레벨 기준 불일치로 지원자 혼란
  - 조직 전체의 학습 속도 저하

**사례 3: 부서 간 표준 불일치**
- **상황:** 프론트엔드 팀, DevOps 팀, 데이터 팀 등 여러 팀 존재
- **문제:**
  - 각 팀이 "시니어" 기준을 다르게 정의
  - 지원자가 "귀사의 시니어 개발자 기준이 뭔가요?" 질문 시 HR도 답변 불가
- **결과:**
  - 지원자 경험 악화
  - 회사 브랜드 이미지 손상 ("이 회사는 기준도 없나?")
  - 내부 공정성 이슈 (같은 직급이 팀마다 다름)

### 고객의 목소리
> "제임스는 채용을 정말 잘해요. 10년 경력이 있고, 면접 때 질문하는 방식도 체계적이에요. 그런데 그게 제임스의 머릿속에만 있어요. 문서화가 안 되어 있으니까 다른 면접관들은 배울 수가 없죠."

> "제임스가 바빠서 면접에 못 오면, 퀄리티가 확 떨어져요."

> "백엔드 팀이 2개가 있는데, 각자 따로따로 채용해요. A팀이 좋은 면접 질문을 만들어도 B팀은 몰라요. 공유할 시스템이 없으니까요."

> "'저희 회사 시니어 개발자가 뭐예요?'라고 물어보면 저도 명확하게 답을 못 해요."

## 현재 해결 방법과 한계

### 현재 시도한 방법
1. **노션 공유 공간 운영**
   - 회사 전체 노션에 채용 관련 페이지 생성
   - 결과: 각 팀이 자기 팀 페이지만 작성, 다른 팀 페이지는 안 봄
   - 한계: 수동 공유 의존, 인센티브 부족

2. **정기 HR 회의**
   - 월 1회 각 팀 채용 담당자 회의
   - 결과: 진행 상황 공유만 하고 노하우 공유는 표면적
   - 한계: 암묵지를 회의에서 전달하기 어려움

3. **제임스에게 문서화 요청**
   - 제임스에게 노하우 정리 요청
   - 결과: 바빠서 미루다가 실행 안 됨
   - 한계: 개인의 추가 업무 부담, 우선순위 낮음

### 왜 실패했는가?
- **공유 인센티브 부족:** 노하우 공유해도 개인/팀에게 돌아오는 이득 없음
- **암묵지 명시화의 어려움:** 제임스의 노하우는 경험과 직관, 문서로 옮기기 어려움
- **사일로 문화:** 팀 간 경계, "우리 팀 방식이 최고" 마인드
- **발견의 어려움:** 좋은 사례가 있어도 찾을 수 없으면 무용지물

## 온톨로지 기반 솔루션 가능성

### HR 온톨로지가 해결할 수 있는 것

**1. 자동 베스트 프랙티스 추출**
```
제임스의 면접 데이터 분석:
- 사용한 질문 20개
- 각 질문의 효과성 (합격자 vs 불합격자 변별력)
- 평가 패턴 (어떤 역량에 높은 가중치)
- 합격자 특성 (평균 점수 분포)

시스템이 자동으로:
- 제임스의 우수 질문 TOP 10 추출
- 다른 면접관에게 추천
- "제임스는 이 질문으로 우수한 후보자를 선별했습니다" 컨텍스트 제공
```

**2. 팀 간 지식 네트워크**
```
백엔드 개발자 채용 (전사)
├─ A팀 노하우
│  ├─ 추천 질문 15개 (사용 빈도 + 효과성 순)
│  └─ 평가 루브릭 (커뮤니케이션: 기술 설명 중시)
├─ B팀 노하우
│  ├─ 추천 질문 12개
│  └─ 평가 루브릭 (커뮤니케이션: 협업 능력 중시)
└─ 공통 베스트 프랙티스
   ├─ 양 팀 모두 효과적이었던 질문 5개
   └─ 통합 루브릭 제안
```

**3. 표준의 자동 수렴**
```
"시니어 백엔드 개발자" 정의:

현재 상태:
- A팀: 경력 5년+ (80% 가중치)
- B팀: 리딩 경험 (70% 가중치)
- DevOps팀: 경력 7년+ (90% 가중치)

시스템 제안:
"세 팀의 합격자 데이터 분석 결과, 공통 요소는:
- 경력 5년+ AND (리딩 경험 OR 멘토링 2년+)
이것을 전사 '시니어' 기준으로 제안합니다."
```

**4. 지식 발견 가능성 (Discoverability)**
```
상황: 프론트엔드 팀이 "코드 리뷰 문화" 평가 방법 고민

시스템 알림:
"백엔드 A팀이 유사한 역량('코드 품질 관리')을 평가했습니다.
- 사용 질문: '코드 리뷰 시 가장 중요하게 보는 것은?'
- 효과성: 4.5/5 (10회 사용)
- 담당자: 제임스 (연락 가능)
이 질문을 참고하시겠습니까?"
```

### 핵심 가치 제안
- **자동 추출:** 개인이 문서화 안 해도 시스템이 패턴 발견
- **문맥 공유:** 단순 질문이 아니라 "왜 효과적이었는지" 컨텍스트 포함
- **발견 가능:** 적절한 순간에 관련 노하우 자동 제안
- **표준 수렴:** 데이터 기반으로 자연스럽게 베스트 프랙티스 도출

## 예상 임팩트

### 정량적 임팩트
- **팀 간 중복 작업 감소:** 30% 감소
  - 근거: 다른 팀의 검증된 방법 재사용
- **신규 팀 온보딩 가속화:** 50% 시간 단축
  - 근거: 기존 팀의 베스트 프랙티스 즉시 학습
- **채용 품질 편차 감소:** 표준편차 40% 감소
  - 근거: 모든 팀이 우수 사례 활용으로 하향 평준화 방지

### 정성적 임팩트
- **조직 학습 문화:**
  - 개인의 노하우가 조직 자산으로 전환
  - "내 방법이 다른 팀에 도움 됐다"는 보람
- **공정성 및 신뢰:**
  - 팀 간 일관된 기준으로 내부 공정성 강화
  - 지원자에게 명확한 메시지
- **전문가 부담 경감:**
  - 제임스가 모든 면접 참석 안 해도 됨
  - 시스템이 제임스의 노하우 전파
- **협업 강화:**
  - 팀 간 "우리 vs 그들" 아닌 "함께 배움"
  - 채용 관련 크로스 팀 대화 증가

### 2차 효과
- **인재 이동 원활화:** 팀 간 기준 일치로 내부 전환 쉬움
- **조직 확장성:** 새 팀 생길 때 기존 지식 즉시 활용
- **브랜드 일관성:** 모든 지원자가 일관된 경험

## 가정 (Assumptions)

### 검증 필요한 가정
1. **공유 의향:**
   - 가정: 팀들이 자기 팀 노하우를 공유할 의향 있음
   - 리스크: "우리 팀 경쟁 우위를 왜 공유하나" 저항
   - 검증 방법: 팀 리더 인터뷰, 인센티브 설계 논의

2. **노하우 추출 가능성:**
   - 가정: 제임스 같은 전문가의 암묵지를 데이터로 추출 가능
   - 리스크: 암묵지는 본질적으로 명시화 불가능
   - 검증 방법: 제임스의 과거 10회 채용 데이터 분석, 패턴 발견 시도

3. **표준 수용성:**
   - 가정: 데이터 기반 표준 제안을 팀들이 수용
   - 리스크: "우리 팀은 특수하다" 주장으로 거부
   - 검증 방법: 표준 제안을 강제 아닌 권장으로 설계, 채택률 측정

4. **크로스 팀 적용 가능성:**
   - 가정: 백엔드 팀의 노하우가 프론트엔드 팀에도 유용
   - 리스크: 직무 차이로 실제론 적용 불가
   - 검증 방법: 유사 역량(예: 커뮤니케이션) 중심으로 먼저 테스트

## 경쟁 솔루션 분석

### 기존 시장 솔루션
1. **위키/컨플루언스:**
   - 제공: 문서 공유 플랫폼
   - 한계: 수동 작성, 발견 어려움, 업데이트 안 됨
   - 차별점: 자동 패턴 추출 + 상황별 추천

2. **HR 컨설팅 (역량 모델 표준화):**
   - 제공: 전사 역량 프레임워크 설계
   - 한계: 일회성, 실행 단계에서 팀별로 다시 분화
   - 차별점: 살아있는 시스템으로 지속 업데이트

3. **Learning Management System (LMS):**
   - 제공: 교육 콘텐츠 공유
   - 한계: 정형화된 교육 자료, 실시간 채용 노하우와 거리
   - 차별점: 실제 채용 과정에 통합, 즉시 활용 가능
