# Opportunity: 채용 지식 재사용

**식별일:** 2025-11-16
**출처:** 보리 인터뷰 (snapshot-borry-2025-11-16.md)
**우선순위:** High
**임팩트 영역:** 업무 효율성, 채용 품질 향상, 조직 학습

---

## 고객 문제 (Customer Problem)

### 구체적 상황
채용 담당자는 **과거에 만든 면접 질문, JD, 평가 방법이 노션에 축적되어 있지만, 맥락 정보가 없어 재사용할 수 없어** 매 채용마다 같은 작업을 처음부터 반복합니다.

### 실제 사례
**노션의 무용지물 자료:**
- **상황:** 노션에 면접 질문 30개 축적
- **문제:** 각 질문에 대한 맥락 정보 부재
  - 언제 만들어진 질문인지
  - 누가 만들었는지
  - 어떤 포지션용인지 (주니어? 시니어? 리드?)
  - 왜 이 질문을 만들었는지 (어떤 역량 평가용?)
  - 과거 이 질문으로 뽑은 사람이 실제로 어땠는지

**구체적 예시:**
- **질문:** "팀 내 갈등을 어떻게 해결했나요?"
- **없는 정보:**
  - 이 질문이 "갈등 조정 능력" 평가용인지, "리더십" 평가용인지
  - 주니어에게도 물어볼 질문인지, 시니어 이상용인지
  - 좋은 답변의 기준이 무엇인지
  - 과거 이 질문에 잘 답한 후보자가 입사 후 성과가 좋았는지

**결과:**
- 질문을 찾았지만 사용하지 못하고 처음부터 새로 작성
- 2시간 소요
- 과거 데이터 검색 시도했으나 노션 페이지 100개+ 중 찾기 어려움
- 결국 기억에 의존

### 고객의 목소리
> "노션에 면접 질문을 30개 정도 모아뒀거든요. 그런데 막상 쓰려고 보니까 이게 언제, 누가, 어떤 포지션을 위해서 만든 질문인지 알 수가 없는 거예요."

> "그래서 결국 안 쓰고 매번 새로 만들게 돼요. 그러면 2시간은 훌쩍 가버리죠."

> "작년에 백엔드 개발자 5명을 뽑았는데, 그때 합격자들이 어떤 스킬을 가지고 있었는지, 어떤 질문에 어떻게 답했는지 찾기가 너무 어려워요."

> "노션 페이지가 100개가 넘는데 검색도 잘 안 되고... 그냥 기억에 의존하게 되죠."

---

## 현재 해결 방법과 한계

### 현재 시도한 방법
1. **노션에 자료 저장**
   - 면접 질문, JD, 평가표를 노션 페이지로 저장
   - 결과: 자료는 쌓이지만 찾기 어렵고 맥락 없음
   - 한계: 질문만 있고, 왜/언제/누구를 위한 것인지 정보 없음

2. **폴더/태그 구조화 시도**
   - 포지션별 폴더 생성
   - 결과: 초기엔 정리되다가 시간 지나면 무너짐
   - 한계: 수동 정리 부담, 일관성 유지 어려움

3. **검색 의존**
   - 노션 검색 기능 사용
   - 결과: 키워드로만 찾아지고 맥락 필터링 불가
   - 한계: "시니어용 커뮤니케이션 질문" 같은 구체적 검색 불가

### 왜 실패했는가?
- **맥락의 부재:** 콘텐츠만 저장하고 메타데이터(누가, 언제, 왜, 어떤 맥락) 저장 안 함
- **수동 정리의 한계:** 바쁜 상황에서 메타데이터 입력은 후순위
- **데이터 연결 부족:** 질문-역량, 질문-포지션, 후보자-성과 연결 안 됨
- **학습 부재:** 과거 데이터에서 배우지 못함 (어떤 질문이 좋은 후보자 선별에 효과적이었는지)

## 온톨로지 기반 솔루션 가능성

### HR 온톨로지가 해결할 수 있는 것

**1. 지식 자산의 구조화된 메타데이터**
```
면접 질문: "팀 내 갈등을 어떻게 해결했나요?"
├─ 평가 역량: 갈등 조정 능력 (hasCompetency)
├─ 적합 포지션: 시니어 이상 (applicableToLevel)
├─ 작성자: 제임스 (createdBy)
├─ 작성일: 2024-03-15 (createdDate)
├─ 사용 이력:
│  ├─ 백엔드 개발자 채용 (2024-05-10)
│  └─ 프론트엔드 리드 채용 (2024-08-22)
├─ 좋은 답변 기준:
│  ├─ 5점: 구체적 상황, 다중 이해관계자, 데이터 기반 해결
│  └─ 3점: 일반적 상황, 개인 간 갈등, 감정적 접근
└─ 효과성 데이터:
   ├─ 이 질문에 4점 이상 받은 후보자 10명
   └─ 입사 후 성과 평균: 4.2/5 (높은 상관관계)
```

**2. 지능형 검색 및 추천**
```
상황: "시니어 백엔드 개발자 채용, 리더십 평가 필요"

시스템 추천:
1. "팀 내 갈등 해결" 질문
   - 이유: 시니어 레벨 + 리더십 역량 평가
   - 과거 사용: 5회, 평균 만족도 4.5/5
   - 효과성: 이 질문 고득점자의 입사 후 성과 우수

2. "기술 결정 시 팀 합의 과정" 질문
   - 이유: 시니어 백엔드 + 의사결정 역량
   - 과거 사용: 3회, 평균 만족도 4.2/5
```

**3. 채용 지식의 연결 네트워크**
```
시니어 백엔드 개발자
├─ 필수 스킬
│  └─ RESTful API 개발
│     └─ 추천 면접 질문 3개 (사용 이력 + 효과성 순)
├─ 필수 역량
│  └─ 리더십
│     ├─ 추천 면접 질문 5개
│     └─ 평가 루브릭 (5단계)
└─ 과거 합격자 프로필
   ├─ 보유 스킬 분포
   ├─ 평균 평가 점수
   └─ 입사 후 성과 분포
```

**4. 지속적 학습 메커니즘**
```
질문 효과성 업데이트 (자동):
- 후보자가 질문에 답변 → 점수 기록
- 합격 후 6개월 성과 리뷰
- 질문 점수 vs 성과 상관관계 계산
- 효과적인 질문은 상위 랭크, 비효과적 질문은 개선 제안
```

### 핵심 가치 제안
- **맥락화된 재사용:** 질문을 찾을 때 "왜/언제/누구용" 자동 표시
- **지능형 추천:** 상황에 맞는 최적 질문 자동 제안
- **학습 루프:** 사용할수록 똑똑해지는 시스템
- **효율성:** 2시간 작업을 15분으로 단축

## 예상 임팩트

### 정량적 임팩트
- **면접 질문 준비 시간 단축:** 2시간 → 15분 (87.5% 감소)
  - 근거: 맥락 기반 검색 및 추천으로 즉시 사용 가능
- **JD 작성 시간 단축:** 2시간 → 30분 (75% 감소)
  - 근거: 유사 포지션 과거 JD 활용
- **채용 품질 향상:** 우수 후보자 선별 정확도 30% 증가
  - 근거: 효과성 검증된 질문 사용
- **지식 재사용률:** 0% → 70%
  - 근거: 현재는 거의 재사용 안 함, 시스템 도입 시 대부분 재사용

### 정성적 임팩트
- **누적 학습:**
  - 채용할 때마다 조직의 채용 지식 축적
  - 시간이 지날수록 더 나은 채용 프로세스
- **신입 담당자 온보딩 가속화:**
  - 과거 베스트 프랙티스를 즉시 학습
  - 제임스 같은 전문가 의존도 감소
- **의사결정 자신감:**
  - "이 질문이 효과적이었다"는 데이터 근거
  - 검증된 방법 사용으로 확신 증가
- **지식 소실 방지:**
  - 담당자 퇴사 시에도 노하우 유지
  - 조직 기억(organizational memory) 구축

### 2차 효과
- **A/B 테스트 가능:** 다른 질문의 효과성 비교
- **채용 트렌드 분석:** 시간에 따른 스킬 수요 변화 파악
- **벤치마크:** 우리 회사 vs 업계 표준 비교

## 가정 (Assumptions)

### 검증 필요한 가정
1. **메타데이터 입력 부담:**
   - 가정: 시스템이 메타데이터 자동 생성/제안하면 부담 최소화
   - 리스크: 여전히 수동 입력 필요해서 사용 안 함
   - 검증 방법: 메타데이터 자동 추론 기능 프로토타입 테스트

2. **과거 데이터 마이그레이션:**
   - 가정: 노션 기존 자료를 온톨로지로 이전 가능
   - 리스크: 맥락 정보 부족으로 이전 불가, 처음부터 시작
   - 검증 방법: 샘플 데이터 10개로 마이그레이션 파일럿

3. **추천 정확도:**
   - 가정: 시스템 추천이 실제로 유용함
   - 리스크: 부적절한 추천으로 신뢰 상실
   - 검증 방법: 과거 채용 데이터로 추천 알고리즘 시뮬레이션

4. **효과성 측정 가능성:**
   - 가정: 질문 점수 vs 입사 후 성과 상관관계 측정 가능
   - 리스크: 변수 너무 많아 인과관계 불명확
   - 검증 방법: 통계적 유의성 분석, 최소 샘플 크기 계산

## 경쟁 솔루션 분석

### 기존 시장 솔루션
1. **노션, Confluence 같은 위키:**
   - 제공: 문서 저장 및 기본 검색
   - 한계: 맥락 메타데이터 관리 약함, 추천 기능 없음
   - 차별점: HR 온톨로지는 구조화된 메타데이터 + 지능형 추천

2. **ATS의 질문 라이브러리:**
   - 제공: 질문 템플릿 저장
   - 한계: 효과성 데이터 없음, 단순 저장소
   - 차별점: 사용 이력 + 효과성 학습 + 상황별 추천

3. **BrightHire, Metaview 같은 인터뷰 인텔리전스:**
   - 제공: 면접 녹화, 전사, 인사이트 추출
   - 한계: 개별 면접 분석, 조직 차원 지식 축적 약함
   - 차별점: 온톨로지로 조직 전체의 채용 지식 네트워크 구축
