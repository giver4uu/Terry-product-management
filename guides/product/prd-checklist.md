## Quick Reference: 필수 vs 권장

### 필수 섹션 (Must Have)

| 섹션 | 없으면 안 되는 이유 |
|------|-------------------|
| **JTBD** | 팀이 같은 문제를 풀고 있는지 확인 |
| **Research Evidence** | "올바른 문제인가" 검증 근거 |
| **Before/After** | 변화의 가치를 명확히 전달 |
| **User Stories + AC** | 개발자가 "완료"를 판단 |
| **Success Metrics** | "성공했는가" 판단 기준 |
| **Key Assumptions** | 리스크 관리 및 사전 대응 |
| **Out of Scope** | 스코프 크립 방지 |

### 권장 섹션 (Nice to Have)

| 섹션 | 있으면 좋은 이유 |
|------|-----------------|
| ASCII Art UI | 개발자/디자이너 커뮤니케이션 효율 |
| Decision Log | 3개월 후 "왜?" 질문에 답변 |
| FAQ | 반복 질문 사전 방지 |
| 용어 정의 | 신규 팀원 온보딩 |

## 점수 기반 자가 평가

### PRD 품질 점수 (총 100점)

| 영역 | 배점 | 자가 평가 |
|------|:----:|:--------:|
| Research Evidence 충실도 | 15점 | /15 |
| JTBD + Pain Points 명확성 | 15점 | /15 |
| 솔루션 설명 (Before/After 등) | 10점 | /10 |
| User Stories + AC 구체성 | 15점 | /15 |
| Success Metrics 측정 가능성 | 15점 | /15 |
| Key Assumptions 검증 계획 | 15점 | /15 |
| 리스크 관리 + Out of Scope | 10점 | /10 |
| 문서 가독성 (주니어 이해 가능) | 5점 | /5 |
| **총점** | **100점** | **/100** |

### 등급 기준

| 등급 | 점수 | 의미 |
|------|------|------|
| A | 90+ | 개발 착수 가능 |
| B | 75-89 | 일부 보완 후 착수 |
| C | 60-74 | 주요 섹션 보완 필요 |
| D | 60 미만 | 재작성 권장 |
